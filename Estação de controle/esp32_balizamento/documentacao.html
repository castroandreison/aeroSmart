4. Tópicos MQTT

O sufixo {estacao} é definido na configuração (padrão: AeroClub Central).

TópicoDireçãoFinalidade
Bal/Write/{estacao}Platforma → ESP32Comandos do balizador
Bal/Read/{estacao}ESP32 → PlataformaStatus, consumo e confirmações

SDM120/Read/{estacao}ESP32 → PlataformaLeituras dos registradores
Heartbeat/{estacao}ESP32 → PlataformaHeartbeat periódico

4.1 Comandos recebidos — Bal/Write/{estacao}

BalOn

Liga o relé da pista.

{"comando": "BalOn"}

Com parâmetros do agendamento:

{
  "comando": "BalOn",
  "agendamento": {
    "id": 1, "data": "2026-07-17",
    "horario": "10:00", "hora_termino": "10:30",
    "duracao_minutos": 30
  }
}

Resposta: publishStatus() → {"comando":"BalOn","estado":true,"timestamp":"..."}

BalOff

Desliga o relé da pista.

{"comando": "BalOff"}

Resposta: publishConsumption() + publishStatus(BalOff:false)

AgendarBalizamento

Agenda uma ativação futura. Armazenado em NVS e executado automaticamente.

{
  "comando": "AgendarBalizamento",
  "agendamento": {
    "id": 77,
    "data": "17/07/2026",
    "hora_inicio": "10:00:00",
    "hora_fim": "10:30:00",
    "duracao_segundos": 1800
  }
}

Validações: id > 0, hora_fim > hora_inicio, end_ts > now

Resposta: publishScheduleStatus(id, "AgendamentoConfirmado", "agendado")

CancelarAgendamento

{"comando": "CancelarAgendamento", "agendamento": {"id": 77}}

RequestHeartbeat

{"comando": "RequestHeartbeat"}

Resposta: publishHeartbeat() em Heartbeat/{estacao}

UpdateFirmware

{"comando": "UpdateFirmware", "url": "https://exemplo.com/firmware.bin"}

4.2 Comandos recebidos — SDM120/Write/{estacao}

ReadRegistersEnergy

{"comando": "ReadRegistersEnergy"}

Resposta: publishEnergyRegisters()

5. API HTTP (WebServer — porta 80)

5.1 Página principal

GET /HTML completo com CSS e JS embarcados.

5.2 Endpoints REST

GET /api/status

Estado atual do dispositivo (relé, WiFi, MQTT, SDM120).

POST /api/control

Liga/desliga o relé via ação web.

// Liga por 15 min
{"action": "on", "duration_min": 15}
// Desliga
{"action": "off"}
// Resposta
{"success": true}

GET /api/history

Histórico de ativações (máx. 10 registros).

{"history": [{"date":"17/07/2026","duration_sec":1800,"energy_kwh":0.6,"completed":true}], "count": 1}

GET /api/schedules

Agendamentos futuros.

{"schedules": [{"id":77,"start":"17/07/2026 10:00:00","end":"17/07/2026 10:30:00","executed":false}], "count": 1}

GET /api/config

Configuração salva em NVS.

POST /api/config

Salva configuração (requer reboot).

// Resposta
{"success": true, "needs_reboot": true}

GET /api/scan

Escaneia redes WiFi disponíveis.

POST /api/ota

Atualização OTA via URL.

{"url": "https://exemplo.com/firmware.bin"}

POST /api/restart

Reinicia o ESP32.

8. Timezone

O ESPHome não chama setenv("TZ")/tzset() em plataformas embarcadas. O firmware chama explicitamente com formato POSIX para que mktime() interprete struct tm como hora local.

Config (IANA)POSIX aplicadoOffset
America/Sao_Paulo (padrão)BRT3UTC-3
America/ManausBRT4UTC-4
America/NoronhaBRT2UTC-2
